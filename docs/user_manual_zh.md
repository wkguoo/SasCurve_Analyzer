# SAS Curve Analyzer 使用手册

本手册面向第一次使用 `SAS Curve Analyzer` 的材料、化学、物理、高分子或相关方向研究生。目标是让你能按照步骤完成一次基础的一维小角散射曲线导入、检查、绘图、无模型分析、批量比较和结果导出。

软件适合处理已经完成一维归约和校准的 SAXS/SANS/WAXS-SAXS 曲线。它不会替你完成二维探测器积分、背景扣除、透过率校正、厚度校正或绝对强度校准。

## 1. 软件简介

### 1.1 这个软件解决什么问题

`SAS Curve Analyzer` 是一个本地 Windows 桌面工具，用来处理一维散射曲线文件。它的核心工作是：

- 导入包含 `q` 和 `I(q)` 的曲线文件；
- 检查 q 排序、重复 q、NaN、零强度、负强度和 error 列异常；
- 绘制 linear、semi-log、log-log、Guinier、Kratky、Porod、invariant、local slope、peak/d-spacing 等视图；
- 运行 Guinier、power-law、local slope、peak detection、finite invariant、Kratky、Porod 等无模型或描述性分析；
- 比较多条曲线，导出 Origin/Excel/Python 可用的数据表和当前曲线第一手转换数据；
- 记录处理历史，方便复查“做过什么、用过什么 q 范围、有什么 warning”。

### 1.2 适用数据类型

适合输入的数据是已经从实验软件或同步辐射/中子平台导出的 1D 曲线：

- 文件格式：`.csv`、`.txt`、`.dat`；
- 必需列：`q` 和 `I(q)`；
- 可选列：`error`、`sigma`、`std`、`uncertainty`；
- q 单位：常见为 `A^-1` 或 `nm^-1`；
- 强度单位：常见为 `cm^-1` 或任意相对强度单位。

### 1.3 不适合用本软件完成的工作

本软件不做以下工作：

- 二维探测器图像积分；
- 背景扣除、空白扣除、透过率校正、厚度校正；
- 绝对强度校准；
- 自动结构识别；
- 自动粒径判断；
- 复杂结构模型拟合的最终物理论证。

如果你的原始数据仍是二维图像，或还没有完成必要校正，应先在专业数据归约软件中处理，再导入本软件。

### 1.4 推荐使用场景

- 组会前快速检查多条 SAS 曲线形状；
- 对已校准 1D 曲线做 Guinier、power-law、peak、Kratky、Porod 等初步分析；
- 批量导入原位或时间序列曲线，确认顺序和基本趋势；
- 导出 Origin 友好长表、矩阵表、feature table 和当前曲线转换数据 CSV；
- 为论文初稿整理可追溯的分析记录。

## 2. 新手快速上手

### 2.1 准备一个曲线文件

准备一个至少包含两列的文本文件，例如：

```csv
q,I,error
0.010,980.2,19.6
0.012,971.6,19.4
0.014,960.1,19.2
```

如果没有误差列，也可以只包含：

```csv
q_A_inv,intensity_cm_inv
0.010,980.2
0.012,971.6
0.014,960.1
```

### 2.2 启动软件

在项目根目录运行：

```powershell
python -m pip install -r requirements.txt
python main.py
```

成功标准：出现标题为 `SAS Curve Analyzer` 的桌面窗口。

### 2.3 导入数据

进入 `数据导入` 页：

1. 点击 `选择数据文件`。
2. 选择 `.csv`、`.txt` 或 `.dat` 文件。
3. 检查自动识别的 `q 列`、`I(q) 列`、`error/sigma 列`、`q 单位` 和 `强度单位`。
4. 查看自动生成的导入前预览/诊断结果。
5. 如果手动修改列名，点击 `预览/诊断当前文件` 重新检查。
6. 点击 `导入曲线`。

成功标准：左侧曲线列表出现新曲线，导入日志显示点数和是否有 error 列。

### 2.4 数据检查

进入 `数据导入` 顶层页中的 `数据检查` 子页，点击检查按钮。重点看：

- q 是否递增；
- 是否有重复 q；
- 是否有 NaN；
- 是否有零强度或负强度；
- error 列是否有负值或非有限值。

Warning 不等于数据不能用，但必须知道它会影响哪些图和分析。

### 2.5 绘制常用图

进入 `曲线工作台`，使用左侧 `曲线绘图` 区域：

1. 在 `Plot type` 选择图类型。
2. 根据需要勾选 `Show error bars`、`Show d = 2π/q axis`、`Annotate q* / d`。
3. 点击 `Plot current curve`。
4. 鼠标移动到图上时，下方独立坐标行会显示当前坐标和反推的 q/I 信息。
5. 需要导出当前图时，选择图像导出 preset 和格式，再点击 `Export current figure`。

### 2.6 选择 q 范围并运行无模型分析

进入 `曲线工作台`，使用右侧 `曲线分析` 区域：

1. 选择分析类型，例如 `Guinier analysis` 或 `Power-law slope`。
2. 点击 `使用当前曲线 q 范围`，再手动缩小 `q_min/q_max`。
3. 如果你在曲线绘图区域已经放大了某个 `ln q` 或 `q²` 区间，可点击 `使用曲线绘图区域 x 范围换算为 q`。
4. 点击 `检查当前 q 范围` 查看点数、log 可用点和过滤事实。
5. 点击 `运行分析`。

成功标准：输出区显示 `analysis_id`、`analysis_type`、`q_range`、结果字段和 warnings。

### 2.7 导出结果

进入 `项目与输出` 顶层页，再进入 `导出报告` 子页。常用导出包括：

- 当前曲线 CSV；
- feature table；
- Origin 长表；
- Origin 矩阵表；
- 当前曲线转换数据 CSV。

### 2.8 判断自己是否操作成功

一次基础分析成功，至少应满足：

- 曲线导入后能在左侧列表中选中；
- 数据检查没有无法理解的严重错误；
- 曲线绘图区域至少能显示 linear 或 log-log 图；
- 曲线分析区域结果中 `q_range` 是你期望的 raw q 范围；
- 导出文件能在目标文件夹中找到；
- 你能说清楚哪些点被 log 图或 log 分析排除了。

如果当前项目有未保存更改，窗口标题会出现 `*`。关闭可见主窗口、打开其它项目或新建项目前，软件会提示是否丢弃未保存更改。

## 4. 输入数据准备

### 4.1 支持的文件格式

支持 `.csv`、`.txt`、`.dat`。文件可以使用逗号、Tab 或空格分隔。注释行可以以 `#`、`;` 或 `//` 开头。

### 4.2 q、I(q)、error/sigma 列要求

必须有 q 和强度列。误差列可选。q 和 I(q) 应为数值，不能混入单位字符串或中文说明。

### 4.3 推荐列名

推荐列名：

- q：`q`、`q_A_inv`、`q_nm_inv`；
- 强度：`I`、`intensity`、`intensity_cm_inv`；
- 误差：`error`、`sigma`、`std`、`uncertainty`。

### 4.4 单位：A^-1、nm^-1、cm^-1

q 单位会影响 d-spacing 和 q 范围解释。软件支持 `A^-1` 与 `nm^-1` 之间的 q 单位转换。强度单位主要用于记录、绘图轴标签和导出，不会自动校准强度。

### 4.5 注释行、分隔符和缺失值

文件中可以有注释行，但数据表头和数据行应清楚。缺失值、NaN 或非数值内容会在导入或检查时触发 warning 或 error。

### 4.6 绝对强度校准后的轻微负值如何处理

背景扣除或绝对校准后，少量轻微负强度可能出现。软件会保留这些值，不会自动截断为 0，也不会加 offset。linear、Kratky、Porod、invariant 等非 log 显示可以保留它们；log 图和 log 分析仍会排除 `I(q) <= 0`。

### 4.7 数据准备检查清单

- q 和 I(q) 列是否存在；
- q 是否为正值；
- q 是否覆盖你的分析区间；
- I(q) 是否为数值；
- error 列是否可选且非负；
- 文件名是否能区分样品和批次；
- 原始文件是否已备份。

## 6. 数据导入详解

### 6.1 导入单条曲线

操作步骤：

1. 在 `数据导入` 页点击 `选择数据文件`。
2. 选择数据文件。
3. 检查自动填入的列名。
4. 如果识别错误，手动修改 `q 列`、`I(q) 列` 或 `error/sigma 列`。
5. 点击 `预览/诊断当前文件`，检查当前列映射下是否可导入。
6. 点击 `导入曲线`。

输出：左侧新增一条曲线，项目历史中新增导入记录。预览/诊断本身不修改原始文件，也不会自动删除坏点、平移强度或修复 q 排序。

### 6.2 批量导入原位/时间序列曲线

点击 `批量导入曲线文件` 后可以选择多个文件。软件会自然排序文件名，例如 frame 10 会排在 frame 9 后面，而不是 frame 1 后面。文件名中的序列信息会尽量解析到 metadata。

### 6.3 自动识别列名失败时怎么办

如果日志显示列名自动识别失败：

1. 打开数据文件确认表头；
2. 手动填写正确列名；
3. error 列没有时留空；
4. 点击 `预览/诊断当前文件`；
5. 如果状态为 OK 或 Warning，再点击 `导入曲线`。

### 6.3.1 导入前预览/诊断怎么看

预览区会显示：

- 文件路径、状态、行数和列名；
- 当前 q、I(q)、error/sigma 列映射；
- q 和 I(q) 的最小/最大值；
- q 或 I(q) 中的 NaN/非数值数量；
- 重复 q 数量；
- q <= 0 和 I(q) <= 0 的数量；
- error/sigma 中的 NaN 或负值数量；
- 前几行数据。

状态含义：

- `OK`：当前列映射下可导入；
- `Warning`：可导入，但后续绘图或分析可能受影响；
- `Error`：当前列映射下无法导入，需要检查列名或文件格式。

### 6.4 导入后如何确认当前曲线

导入成功后，左侧曲线列表会出现曲线名称和 q 单位。点击该曲线后，检查、绘图、分析等页会使用它作为当前曲线。

### 6.5 常见导入失败原因

- 文件为空；
- 列名写错；
- 数据中混入文字；
- 分隔符不规则；
- q 或 I(q) 列无法转为数值；
- error 列存在负值或非有限值。

## 7. 数据检查详解

### 7.1 检查项含义

数据检查不会修改数据，只给出 warning 或 summary。你应把它当作“分析前体检”。

### 7.2 q 排序、重复 q、NaN、error 异常

- q 未递增：某些积分、峰宽或插值逻辑需要排序；软件内部会在需要时使用排序副本，但你仍应知道原始数据情况。
- 重复 q：可能来自拼接或导出错误。
- NaN：会被过滤或导致分析失败。
- error 异常：负 error 没有物理意义，会影响误差棒和加权拟合。

### 7.3 轻微负强度与显著负强度

轻微负强度可能来自校正后的噪声。显著负强度通常需要回到实验归约流程检查背景、空白、透过率或校准。

### 7.4 为什么 log 图和 log 分析会排除 I(q) <= 0

因为 `ln I(q)` 对零或负数没有实数定义。软件不会为了让 log 图看起来完整而自动加 offset。

### 7.5 如何根据 warning 决定是否继续分析

建议：

- 先看 linear 图，确认原始强度形状；
- 如果 log 图排除点很多，谨慎做 log-based 分析；
- 把所有重要 warning 写进实验记录或报告；
- 不要把 warning 直接解释成材料机理。

## 8. 曲线工作台：曲线绘图详解

### 8.1 线性图 I(q) vs q

最接近原始数据。适合检查负强度、异常尖峰、整体曲线是否导入正确。

### 8.2 Semi-log 图 ln I(q) vs q

显示 `ln I(q)` 随 q 的变化。`I(q) <= 0` 会被排除。

### 8.3 Log-log / power-law 图 ln I(q) vs ln q

适合观察幂律区。注意：当 `0 < q < 1` 时，`ln q` 是负数，这是正常的显示坐标，不代表 q 为负。

#### 8.3.1 双对数图可以回答什么

如果某个 q 区间近似满足：

```text
I(q) = A q^(-alpha)
```

那么双对数图中该区间近似为直线：

```text
ln I(q) = ln A - alpha ln q
```

直线斜率为 `-alpha`。因此双对数图主要用于：

- 判断曲线是否包含一个或多个近似幂律区；
- 估计幂律指数 `alpha`，并检查该指数是否随帧、温度或时间变化；
- 观察峰、肩峰、低 q 上翘以及不同斜率区之间的过渡；
- 判断高 q 是否逐渐转入平坦背景、负强度或噪声主导区；
- 为 power-law、surface-fractal、Porod-like 等后续分析选择候选 q 区间。

双对数图不能单独提供或证明：

- Guinier `Rg`；`Rg` 应在 `ln I(q)` 对 `q²` 的 Guinier 图中评估；
- 唯一的颗粒形貌、孔结构或相组成；
- `d = 2π/q*` 一定等于颗粒直径；
- 某个峰位或指数变化一定由相变或特定形成机理造成；
- 一段看起来近似直线的短区间一定具有物理幂律意义。

判断一个幂律结果是否可用时，至少同时检查：拟合 q 区间、点数、log-q 跨度、`R²`、局部斜率稳定性、负强度过滤情况以及是否受到峰或背景影响。不要只看 `R²`。

#### 8.3.2 Ti15 前十帧实例：双对数图能够提取什么

本例使用 `results/spectra_csv` 中的：

```text
ti15_00001_abs2d_cm-1.csv
...
ti15_00010_abs2d_cm-1.csv
```

每帧包含 5500 个 q-I(q) 点，q 范围约为 `9.35×10^-5–1.028 A^-1`。每帧有 4277–4405 个正强度点和 1084–1217 个负强度点。双对数图及 log-based 分析只使用 `q > 0`、`I(q) > 0` 的有限点，不会给负强度加常数或将其截断为零。

双对数图可以把本例大致分为以下区域：

| 区域 | 数据表现 | 可获得的信息 | 当前软件状态 |
|---|---|---|---|
| 极低 q | 波动并总体向主峰方向上升 | 低 q 数据质量、是否存在上翘或稳定低 q 衰减 | 可以检查风险；本例没有可用 Guinier/低 q 幂律 |
| 主峰 | 自动峰区约 `0.008354–0.011376 A^-1` | 峰位、峰宽、强度、面积和特征间距 | 可以稳定提取 |
| 峰后衰减 | 共识区 `0.013178–0.021963 A^-1` | 幂律指数及幂律类型候选 | 10/10 帧可以稳定提取 |
| 中高 q 过渡 | 斜率逐渐减小、曲线趋平 | 衰减向背景主导区的变化 | 图上可以观察；唯一 crossover 尚不能可靠自动报告 |
| 高 q 尾部 | 背景平台、负值和强噪声 | 建议排除的风险区 | 可以标记风险，不适合继续做 log 拟合 |

##### 稳定幂律结果

自动处理在十帧中得到共同幂律区：

```text
q = 0.013178–0.021963 A^-1
```

该共同区间每帧包含 48 个有效点。逐帧拟合结果为：

- `alpha = 3.399–3.488`；
- 十帧平均 `alpha ≈ 3.439`；
- `R² = 0.9972–0.9993`；
- 10/10 帧均成功计算。

因此，这批数据在峰后具有稳定的近似 `I(q) ∝ q^-3.44` 衰减。这是本例双对数图中最可靠的定量信息。

因为 `3 < alpha < 4`，它可以作为表面分形型散射候选。在相关散射假设成立时，可计算候选表面分形维数：

```text
Ds = 6 - alpha ≈ 2.51–2.60
```

这里的 `Ds` 只能作为模型条件下的候选量。软件稳定提取的是 `alpha`；只有在材料体系、背景处理、适用 q 区间和其他表征共同支持时，才能进一步引用 `Ds`，不能把它单独作为表面分形形貌的证明。

##### 主峰与特征间距

十帧均检测到：

```text
q* = 0.00925269 A^-1
d = 2π/q* ≈ 679.1 A ≈ 67.9 nm
```

十帧峰位落在同一个离散 q 点。当前 q 采样间隔约为 `1.87×10^-4 A^-1`，所以正确表述是“在当前 q 分辨率内没有可分辨峰位移动”，而不是“峰位绝对不变”。`d` 是特征相关尺度，不自动等于颗粒直径。

##### 为什么不是经典 Porod 区

本例峰后共同指数约为 `3.44`，没有达到经典 Porod 所需的 `alpha ≈ 4`。随着 q 增大，斜率还在继续减小。以十帧中位曲线作描述性比较：

- `0.020–0.040 A^-1`：`alpha ≈ 2.9`；
- `0.040–0.100 A^-1`：`alpha ≈ 0.77`；
- `0.100–0.400 A^-1`：`alpha ≈ 0.05`。

这表明曲线逐渐转入近似平坦的背景，而不是形成稳定的 `q^-4` 尾部。软件把约 `q >= 0.321 A^-1` 标为高 q 噪声/背景风险区，该区每帧非正强度比例约为 29%–32%。这个边界是自动风险提示，不应解释成精确的物理背景起点。

##### 为什么没有 Guinier 结果

Guinier 分析需要低 q 区满足 `ln I(q)` 对 `q²` 近似线性、斜率为负，并满足 `qRg` 等条件。本例极低 q 强度波动并向主峰上升，没有形成稳定的 Guinier 单调衰减。软件已经扫描这些低 q 数据，但没有通过方法判据。因此，“没有 Rg”表示当前测量范围中没有可辩护的 Guinier 区，不表示这一帧没有信息。

##### 当前软件可以与暂时不能可靠提取的项目

| 项目 | 当前结论 |
|---|---|
| 峰位、峰区和 `d = 2π/q*` | 可以成功提取 |
| 共同幂律区和逐帧 `alpha` | 可以成功提取 |
| 高 q 噪声/背景风险 | 可以标记 |
| 表面分形维数 `Ds` | 只能在相关模型假设成立时条件性换算 |
| 逐点 local slope `alpha(q)` | 可以计算，但本例标准差约为 `0.81–1.30`，只找到 0–2 个很短的平台；暂不宜作为主要定量结论 |
| 全曲线 crossover | 本例噪声会产生大量数值候选，当前版本不能可靠报告唯一 crossover q |
| Guinier `Rg` | 已扫描但判据不满足，正确留空 |
| 经典 Porod 指数/平台 | 已扫描但判据不满足，正确留空 |

“每帧都有有效信息”不等于“每帧都必须产生 Rg、幂律指数和 Porod 指数”。对于不适用的方法保留空值和原因，是科学分析结果的一部分，不能通过降低阈值或强行拟合补齐。

## 9. q 范围与坐标变换

### 9.1 物理 q 不能为负

SAS 中 q 是散射矢量模长，物理 q 范围应为正。曲线分析区域的 `q_min/q_max` 始终表示 raw q。

### 9.2 为什么 ln q 可以为负

当 `0 < q < 1` 时，`ln q < 0`。这只是图上横坐标变换后的结果，不代表 q 是负数。

### 9.3 图上 x 范围和分析 q 范围的区别

- raw q 范围：真正传给分析函数的物理区间；
- display x 范围：当前图上的横坐标，可能是 q、ln q 或 q²。

### 9.4 如何把当前图 x 范围转换为 raw q 范围

在曲线绘图区域调整好当前 X 范围后，回到 `曲线工作台` 的 `曲线分析` 区域，点击 `使用曲线绘图区域 x 范围换算为 q`。软件会根据当前 plot type 自动换算：

- raw q 图：`q = x`；
- log-log 或 log-q contribution 图：`q = exp(x)`；
- Guinier 图：`q = sqrt(x)`。

### 9.5 Guinier 图 q² 范围如何换算回 q

Guinier 图横坐标是 q²。例如你看到 x 范围是 0.0001 到 0.0025，那么 raw q 是 0.01 到 0.05。

### 9.6 选择 q 范围的经验原则与风险

软件可以帮你换算坐标，但不能自动判断最佳物理区间。Guinier 区、幂律区、Porod 区都需要结合样品体系、数据质量、重复实验和文献背景判断。

### 9.7 如何理解“已经扫描到，但方法判据不满足”

当结果中没有 Guinier 或 Porod 数值时，不能立刻判断为“软件没找到”。自动区域识别包含两个不同层次：

| 判断层次 | 要回答的问题 | 没有通过时代表什么 |
|---|---|---|
| 第一层：候选区间扫描 | 软件是否检查了这一帧中可能属于低 q、峰区、峰后衰减和高 q 尾部的 q 区间？ | 可能存在扫描范围、窗口设置、数据导入或算法覆盖问题 |
| 第二层：方法判据验证 | 已经扫描到的候选区间是否满足 Guinier、power-law 或 Porod 的物理与数值条件？ | 该方法在当前数据和 q 范围内不适用，结果应留空 |

因此，“扫描不到”和“扫描到了但不接受”不是同一件事：

- **扫描不到**：一个可能有规律的区间没有进入候选列表。例如旧版在宽 q 范围、线性 q 网格上按行号大步跳跃，曾经漏过 Ti15 峰后的稳定幂律段。这属于处理器覆盖问题。
- **扫描到了但不接受**：候选区间已经被计算和检查，但没有通过该方法的适用条件。这属于科学判据结果，不应通过降低阈值强行补数值。

#### 9.7.1 Guinier 为什么可能“扫描到了但不接受”

Guinier 分析要求低 q 区近似满足：

```text
I(q) = I0 exp(-Rg²q²/3)
```

对应的必要检查包括：

- `ln I(q)` 对 `q²` 近似线性；
- 拟合斜率为负，能够得到有限、实数的 `Rg`；
- `qRg` 不超过所采用的经验上限，软件默认重点检查 `qRg_max <= 1.3`；
- 区间具有足够的有效点；
- 区间没有被主峰、低 q 上翘、挡光片边缘或明显波动主导。

如果低 q 强度向某个主峰上升，或者 `ln I(q)` 对 `q²` 没有稳定负斜率，即使程序已经逐个检查低 q 窗口，也不应输出 `Rg`。此时正确结论是：

> 当前测量 q 范围内没有观察到可辩护的 Guinier 区。

这不等于“该帧没有信息”，也不等于“材料不存在某种尺寸结构”；它只说明当前数据不能用 Guinier 近似可靠地测量 `Rg`。

#### 9.7.2 Porod 为什么可能“扫描到了但不接受”

经典 Porod 区通常要求高 q 尾部近似满足：

```text
I(q) ∝ q^-4
```

软件需要同时检查：

- log-log 指数 `alpha` 是否接近 4；
- `q⁴I(q)` 是否形成有限、正值且相对稳定的平台；
- 候选区是否位于可用高 q 部分；
- 候选区是否与负强度、背景平台或高 q 噪声风险区重叠；
- 区间点数和 log-q 跨度是否足够。

只有 `q⁴I(q)` 在一小段内看起来波动较小，并不足以证明 Porod 行为。如果指数明显偏离 4，或高 q 已由背景和噪声主导，程序应拒绝 Porod 结果。此时正确结论是：

> 当前测量 q 范围内没有观察到可靠的经典 Porod 区。

#### 9.7.3 以 Ti15 前十帧为例

修复后的自动处理在十帧中同时识别出：

- 10/10 帧共有的主峰区；
- 10/10 帧共有的峰后幂律区 `0.013178–0.021963 A^-1`；
- 幂律指数 `alpha = 3.399–3.488`，`R² = 0.9972–0.9993`；
- 高 q 背景/噪声风险区。

这些结果证明处理器已经覆盖低 q、峰区、峰后衰减和高 q 尾部。Guinier 和 Porod 仍然留空，是因为：

- 低 q 没有稳定的 Guinier 负斜率和合格 `qRg` 区间；
- 峰后稳定指数约为 `3.44`，不是经典 Porod 的 4；
- 更高 q 逐渐转入平坦背景、负强度和噪声。

所以，本例中“没有 Guinier/Porod”不再是处理器漏扫，而是方法判据没有满足。完整的双对数数据说明见 [8.3.2 Ti15 前十帧实例](#832-ti15-前十帧实例双对数图能够提取什么)。

#### 9.7.4 为什么每帧有意义，却不要求每种方法都有数值

一帧曲线可以通过峰位、峰宽、幂律指数、积分量、背景风险和随序列变化等方式提供有效信息，但不一定同时存在 Guinier 区、幂律区和 Porod 区。

因此：

```text
每帧有信息 ≠ 每帧必须产生所有方法的参数
```

对不适用的方法保留空值、warning 和失败原因，是防止伪结果的重要设计。不能把缺失的 `Rg` 或 Porod 指数用其他区间、前一帧数值或降低阈值自动填补。

## 10. 曲线工作台：曲线分析详解

### 10.0 分析前 q 范围预检

点击 `检查当前 q 范围` 可以在运行分析前查看预检摘要。运行 `运行分析` 时也会自动预检一次。

预检会显示：

- 当前分析类型；
- raw q 范围和范围来源；
- 范围内总点数；
- 有限数值点数；
- 正 q 点数；
- 正强度点数；
- log 可用点数；
- 被排除点数；
- severity：`ok`、`warning` 或 `error`；
- messages。

如果 severity 为 `error`，软件不会继续运行分析。常见原因包括 `q_min >= q_max`、raw q 为负、范围内没有点、点数不足、log 分析可用点不足。预检只判断当前范围是否基本可运行，不会自动判断 Guinier 区、Porod 区或峰区是否科学最优。

### 10.1 Guinier analysis

拟合：

```text
ln I(q) = ln I(0) - Rg² q² / 3
```

输出 Rg、I(0)、拟合统计量和 warning。注意 qRg 条件和线性区选择，不能只看 R2。

### 10.2 Power-law slope

拟合：

```text
ln I(q) = ln prefactor - alpha ln q
```

输出 exponent `alpha`。它是描述性指标，不唯一决定结构。

Ti15 前十帧的完整示例、实际 q 区间、`alpha`、`R²` 和当前软件能力边界见 [8.3.2 Ti15 前十帧实例](#832-ti15-前十帧实例双对数图能够提取什么)。

### 10.3 Local slope

计算：

```text
α(q) = -d ln I / d ln q
```

用于判断幂律区是否稳定。

Local slope 是数值导数，对相邻点噪声和线性 q 网格非常敏感。应优先使用较宽、具有足够 log-q 跨度的整体 power-law 拟合确认指数，再把 `alpha(q)` 当作辅助诊断；不要因为出现一个很短的平台就直接赋予结构含义。

### 10.4 Peak detection

输出峰位、峰强、FWHM、面积和 `d = 2π/q*`。峰位变化可以作为趋势线索，但不能自动解释为某种相变或颗粒尺寸。

### 10.5 Finite invariant

计算有限 q 范围内：

```text
∫q²I(q)dq
```

不包含 q=0 和 q=∞ 外推。

### 10.6 Information budget

用 `q³I(q)` over `ln q` 看 finite invariant 信号集中在哪些 q/d 尺度。

### 10.7 Kratky metrics

从 `q²I(q)` 提取描述性指标。解释依赖材料体系。

### 10.8 Porod metrics

从 `q⁴I(q)` 提取平台统计量。没有 contrast 和高 q 稳定区间时，不应报告绝对表面积。

Porod 判定不能只看某一小段 `q⁴I(q)` 波动较小，还应同时检查 log-log 指数是否接近 4、平台是否为正且稳定、区间是否位于可用高 q 部分，以及是否与高 q 噪声/背景风险区重叠。

### 10.9 从分析页查看对应图

点击 `查看对应图`，软件会切回匹配的绘图类型并刷新图。例如 `Power-law slope` 会切到 log-log 图。

### 10.10 结果中的 warnings、structured warnings 和 reliability 信息

结果中的 warning 是分析结果的一部分，应保存和报告。Warning 的目的不是替你下结论，而是提醒当前结果的使用条件和风险。

## 11. 高级功能：批量比较与原位序列

### 11.1 什么时候使用批量导入

如果你有一组同系列文件，例如原位拉伸、退火、时间序列或温度序列，可以使用批量导入。

批量导入后，`批量比较` 页的序列管理表会显示：

- `sequence_order` 和 `project_order`；
- 曲线 ID 和曲线名；
- 原始文件路径或 source stem；
- `series_id`、`frame_label`、`frame_index`；
- q 单位、强度单位；
- 点数、q_min、q_max；
- warnings，例如 q grid 与第一条曲线不一致、非有限强度或非正强度。

这张表只帮助你复核序列，不会自动做相变、结晶、聚集或动力学解释。

### 11.2 曲线分组

在 `批量比较` 页多选曲线，填写组名，点击 `将选中曲线建组`。分组会写入项目状态和历史记录。

也可以在序列管理表中选中行，然后点击 `从选中行建组`。这种方式会按表格当前序列顺序创建组。

### 11.3 平均曲线

选中至少两条曲线，点击 `平均选中曲线`。软件会生成新曲线，不修改原始曲线。

### 11.4 A/B 比较

选择曲线 A、曲线 B 和比较类型：

- `Difference B - A`；
- `Ratio B / A`；
- `Relative difference (B - A) / A`。

比较结果会保存到项目中。

### 11.5 common q grid 与插值

曲线 q 网格不一致时，比较或平均可能使用共同 q 范围上的插值。插值是为了数值比较，不应被理解为新增实验数据。

### 11.6 导出 Origin-ready long table 和 matrix table

`导出 Origin 长表` 适合大多数批量曲线；`导出 Origin 矩阵表` 需要 q grid 一致。q grid 不一致时，矩阵表可能不生成，并写入 warning。

如果只想复核导入顺序和 frame 信息，可点击 `导出序列索引 CSV`，生成 `sequence_index.csv`。它适合发给导师或自己半年后回查“哪些文件被导入、顺序如何、q 范围和 warning 是什么”。

## 12. 项目、历史记录、正式记录与导出

### 12.1 历史记录和正式记录有什么区别

历史记录记录软件操作，例如导入、分析、比较、导出。正式记录是你主动标记为报告或归档需要重点保留的记录。

### 12.2 保存项目

优先使用菜单栏 `项目` → `保存项目` 或 `另存为项目...`。项目文件夹包含 `project.json` 和内部曲线数据。`导出报告` 页不再提供项目另存为入口，避免把项目管理和数据导出混在一起。

每次保存项目都会写入一条 `project_save` 历史记录，用于追踪项目保存位置和保存时间。这个记录属于项目审计轨迹，不会修改原始实验文件。

打开已有项目时，使用 `项目` → `打开项目...` 并选择包含 `project.json` 的文件夹。打开后左侧曲线列表、分析结果、比较结果、历史记录和正式记录会恢复到保存时的项目状态。

### 12.3 导出曲线 CSV

适合在 Origin、Excel 或 Python 中继续处理单条曲线。

### 12.4 导出分析 JSON/CSV

JSON 适合完整保存结构化结果；CSV 适合表格查看和论文图表整理。

### 12.5 导出当前曲线转换数据 CSV

如果你的原始文件只有 `q` 和 `I(q)` 两列，可以在 `项目与输出` → `导出报告` 中使用：

- `导出当前曲线转换数据 CSV...`：生成 `<curve_name>_transformed_data.csv`。

这个宽表保留每个原始点，一行对应原始输入的一行，不排序、不去重、不删除 NaN/inf 行、不平移负强度、不自动加常数，也不做拟合。某些数学变换无定义时，对应列写空值/NaN；软件不会因为 `log` 或 `2π/q` 无定义而删除原始行。

当前导出列包括：

| 列名 | 公式 | 说明 |
| --- | --- | --- |
| `q` | 原始 q | 不改值 |
| `I(q)` | 原始 I(q) | 不改值 |
| `q²` | q × q | Guinier 横轴的基础变换 |
| `ln q` | ln(q) | 仅 `q > 0` 有定义 |
| `log10 q` | log10(q) | 仅 `q > 0` 有定义 |
| `ln I(q)` | ln(I) | 仅 `I(q) > 0` 有定义 |
| `log10 I(q)` | log10(I) | 仅 `I(q) > 0` 有定义 |
| `q²I(q)` | q² × I(q) | Kratky / finite invariant integrand 的基础列 |
| `q⁴I(q)` | q⁴ × I(q) | Porod 图基础列 |
| `qI(q)` | q × I(q) | 补充诊断列 |
| `q³I(q)` | q³ × I(q) | log-q contribution 的基础列，不代表恢复主图类型 |
| `d = 2π/q` | 2π/q | 仅 `q > 0` 有定义 |

`ln` 是自然对数；`log10` 是 10 为底对数。`q²I(q)` 和 `q⁴I(q)` 只是转换列，不等价于完整物理结论。该 CSV 不包含 alpha、Rg、D、R 或 reference curve 相关参数列。

### 12.6 导出图像

在 `曲线绘图` 页使用 `Export current figure` 导出当前正在查看的图。它会保留当前 plot type、当前曲线、坐标范围、误差棒和辅助轴设置，并应用所选图像预设。导出后仍建议在 Origin、Illustrator 或论文排版软件中做最终排版检查。

### 12.7 输出文件夹结构说明

导出报告页当前只保留稳定、高频、可复核的导出入口：当前曲线 CSV、feature table、Origin 长表、Origin 矩阵表和当前曲线转换数据 CSV。Markdown 报告、完整分析包和项目另存为不再作为该页按钮展示。

## 14. 设置与模型/公式目录

### 14.1 查看当前设置

点击菜单栏 `设置`。可以查看默认 q 单位、图像格式、误差棒、warning 显示和设置文件路径。

### 14.2 默认 q 单位、图像格式、误差棒、导出目录

这些设置影响新导入或新绘图的默认行为，但不会自动修改已导入的原始数据。

### 14.3 轻微负强度阈值设置

可配置是否允许轻微负强度，以及相对幅度和比例阈值。阈值越宽，越可能隐藏真实数据质量问题；正式报告中应说明使用的阈值。

### 14.4 查看计算模型与公式

设置对话框中可打开模型/公式目录，查看绘图、无模型分析、形状模型、P(r)、correlation 和 extrapolation 接口的公式、输入、输出、假设和限制。

### 14.5 如何理解 model-free、descriptive、experimental、reserved、model-dependent

- model-free：较少依赖结构模型，但仍依赖 q 范围和数据质量；
- descriptive：描述形状或趋势，不自动给出机理；
- experimental：实验性接口，不应用于正式结论；
- reserved：预留接口，当前不完整；
- model-dependent：依赖模型假设，拟合好不等于模型唯一正确。

## 15. 高级与实验性功能

### 15.1 P(r) 接口

P(r) 是间接傅里叶变换相关功能，当前属于实验或预留接口。正式使用前需要验证参数、Dmax 和正则化假设。

### 15.2 correlation function 接口

相关函数接口用于未来高级分析，不应直接作为正式结构结论。

### 15.3 extrapolation 接口

低 q 或高 q 外推会显著影响 invariant 和物理量估算。当前不应把预留外推接口当作生产级计算。

### 15.4 shape/form-factor 与经验模型说明

形状模型和经验模型需要材料背景支持。拟合曲线看起来接近，不代表结构唯一。

### 15.5 为什么不能把实验性功能作为正式定量结论

实验性功能可能缺少充分测试、误差估计、边界条件检查或文献验证。正式报告应优先使用已验证的分析路径。

### 16.1 单条已校准 SAXS 曲线的基础分析

## 17. 常见问题与排错

### 17.0 错误提示格式

导入、绘图导出、分析、导出报告和项目保存/打开等常见失败会使用分层事实格式：

- `级别`：信息、警告或错误；
- `发生了什么`：本次操作的客观结果；
- `是否影响原始数据`：说明原始实验文件是否被修改；
- `相关事实`：当前曲线、q 范围、文件夹、计数或状态；
- `技术细节`：底层异常类型和异常文本。

这类提示不包含操作指导或推测原因栏目。

### 17.1 软件打不开

检查：

- 是否安装 Python；
- 是否安装 `requirements.txt` 中依赖；
- 是否在正确项目目录运行；
- 命令行是否显示 PySide6 相关错误。

### 17.2 导入后曲线为空

相关事实：

- 列名选择错误；
- 数据行无法解析为数值；
- 文件分隔符异常；
- 文件实际没有数据行。

### 17.3 找不到 q 或 intensity 列

手动填写列名。列名必须和文件表头一致，大小写通常可自动处理，但特殊字符仍建议避免。

### 17.4 log 图点数变少

log 图会过滤 `q <= 0` 或 `I(q) <= 0`。先回到 linear 图查看原始数据，再决定是否继续做 log-based 分析。

### 17.5 q_min/q_max 不知道怎么选

先在曲线绘图区域放大你想看的区域，再使用 `使用曲线绘图区域 x 范围换算为 q`，或手动根据样品和文献选择 raw q 范围。

### 17.6 为什么 ln q 是负数但 q 不能输入负数

`ln q` 是显示坐标，q 是物理散射矢量模长。q 本身必须为正。

### 17.7 结果看起来不合理

检查：

- q 单位是否正确；
- q 范围是否选错；
- log 分析是否过滤了很多点；
- warning 是否提示点数不足或方法假设不满足；
- 输入数据是否已经完成必要校正。

### 17.8 导出文件在哪里

看导出页输出日志，或检查你选择的导出目录。建议每次分析建立独立输出文件夹。

### 17.9 为什么每帧都有信息，却没有 Guinier 或 Porod 结果

“每帧有信息”表示曲线中仍可提取峰、幂律、积分量、数据质量和序列变化等内容，不代表每一种分析方法都适用。自动处理分为两个问题：

1. 软件是否扫描到了相应 q 区间；
2. 扫描到以后，该区间是否满足对应方法的物理和数值判据。

如果候选区间已经被扫描，但 Guinier 斜率、`qRg`、Porod 指数、`q⁴I(q)` 平台或高 q 噪声判据没有通过，软件应保留空值和失败原因。强行降低阈值只会产生看似完整但不可引用的结果。两层判断的完整解释见 [9.7 如何理解“已经扫描到，但方法判据不满足”](#97-如何理解已经扫描到但方法判据不满足)，具体数据实例见 [8.3.2 Ti15 前十帧实例](#832-ti15-前十帧实例双对数图能够提取什么)。

## 19. 术语表

- q：散射矢量模长，通常单位为 `A^-1` 或 `nm^-1`。
- I(q)：散射强度。
- error/sigma：强度误差或标准差。
- raw q：原始物理 q 坐标。
- display x：当前图上的横坐标，可能是 q、ln q 或 q²。
- Guinier：低 q 近似分析，用于估算 Rg 和 I(0)。
- Rg：回转半径。
- Power-law：幂律斜率分析。
- Kratky：`q²I(q)` 表示。
- Porod：常见为 `q⁴I(q)` 表示。
- invariant：散射不变量的有限 q 范围积分。
- q*：峰位。
- d-spacing：常用 `d = 2π/q*` 估算的特征尺度。

## 20. 方法边界与引用建议

写报告或论文时应说明：

- 数据来源和校正状态；
- q 和强度单位；
- 分析 q 范围；
- log 图或 log 分析排除了哪些点；
- 是否使用轻微负强度阈值设置；
- 每个结果的 warning；
- d、Rg、alpha、Porod plateau 等结果的解释边界。

不要写“软件证明样品为某结构”。更严谨的说法是“在选定 q 范围内，该曲线呈现某种描述性特征，需要结合其它表征和模型验证”。

当前版本顶层只保留四个工作区：`数据导入`、`曲线工作台`、`高级功能`、`项目与输出`。`数据检查` 在 `数据导入` 内，`曲线绘图` 和 `曲线分析` 在 `曲线工作台` 内同屏显示，`深度分析` 和 `批量比较` 在 `高级功能` 内。

导出报告页当前不再提供完整分析包、Markdown 报告、项目另存为、派生长表或派生矩阵表入口；需要复核拟合残差时，以当前分析输出和图形检查为准。
